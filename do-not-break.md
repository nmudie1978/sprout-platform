# Do Not Break — Endeavrly

## Purpose of This File

This file protects critical product qualities, UX standards, architecture decisions, and brand principles during rapid development.

Claude should treat these as high-priority guardrails.

When making changes:

- Preserve what already works
- Avoid regressions
- Avoid accidental dilution of product identity
- Improve carefully, not destructively

If uncertain, choose safer incremental changes.

---

# Core Product Identity (Do Not Break)

Endeavrly is a calm, trusted youth guidance platform.

It must never drift into:

- Generic job board
- Social media clone
- Loud startup gimmick
- Personality-test spam app
- Corporate HR portal
- School admin software
- Gig-economy exploitation platform
- Entertainment-first content feed

Protect identity at all times.

---

# Core User Feeling (Do Not Break)

Users should feel:

- Less lost
- More hopeful
- More capable
- More informed
- Less pressured
- More motivated
- Clearer about next steps

Any feature that increases confusion, anxiety, clutter, or pressure should be challenged.

---

# Sacred Product Areas

## 1. My Journey

This is the emotional core of the product.

Do not break:

- Discover → Understand → Clarity flow
- Sense of progression
- Calm pacing
- Reflection moments
- Personal relevance
- Completion motivation

Changes here must improve the journey, not randomize it.

---

## 2. Dashboard

Do not turn dashboard into chaos.

Protect:

- Simplicity
- Clear next actions
- Useful summaries
- Motivation signals
- Personal relevance
- Fast scanning

No widget overload.

---

## 3. Career Discovery

Protect:

- Easy browsing
- Smart filtering
- Useful comparisons
- Low-friction exploration
- Mobile usability

Do not bury discovery under complexity.

---

# UX Quality (Do Not Break)

## Visual Standards

Do not regress into:

- Cramped layouts
- Poor spacing
- Inconsistent styling
- Weak hierarchy
- Cheap looking UI
- Random colors
- Cluttered screens

Maintain premium polish.

## Interaction Standards

Do not break:

- Responsive feel
- Clear CTAs
- Smooth transitions
- Predictable behavior
- Fast load perception

No awkward friction.

---

# Mobile Experience (Critical)

Most users may use mobile first.

Do not break:

- Thumb usability
- Readability
- Spacing
- Scroll flow
- Sticky CTA logic
- Performance on average devices

Desktop improvements must not damage mobile.

---

# Brand Voice (Do Not Break)

Never introduce copy that feels:

- Corporate
- Cringe
- Fake motivational
- Pushy
- Condescending
- Overly academic
- Salesy

Keep tone:

- Calm
- Clear
- Human
- Respectful
- Modern
- Encouraging

---

# Technical Guardrails

Do not break:

- Existing working flows
- Authentication
- Routing
- Database integrity
- Reusable component structure
- Theme consistency
- Accessibility basics
- Performance unnecessarily

Prefer surgical changes over reckless rewrites.

---

# Design System Integrity

Protect consistency in:

- Buttons
- Inputs
- Cards
- Modals
- Spacing scale
- Typography scale
- Icon usage
- Color logic

No one-off messy components unless strategically justified.

---

# Content Integrity

Do not introduce:

- Fake data
- Overhyped salaries
- Misleading career claims
- Unrealistic promises
- Thin placeholder junk content

Users must trust information quality.

---

# Growth Traps (Avoid)

Do not add growth mechanics that damage trust:

- Spam notifications
- Fake urgency
- Dark patterns
- Manipulative streaks
- Vanity badges
- Endless popups
- Forced sharing prompts

Trust > growth hacks.

---

# Founder Intent (Protect)

This product exists to genuinely help young people navigate uncertainty.

Do not optimize away the soul of the product.

Protect:

- Humanity
- Calmness
- Usefulness
- Honesty
- Dignity
- Long-term trust

---

# When Editing Existing Features

Before changing anything ask:

1. What currently works here?
2. What could regress?
3. Does this improve clarity?
4. Does this improve trust?
5. Does this improve user momentum?
6. Is there a simpler safer path?

---

# Red Flag Triggers

Pause and rethink if a change causes:

- More clutter
- More steps
- More confusion
- Lower trust
- Worse mobile UX
- Slower performance
- Generic feel
- Loss of product uniqueness

---

# Claude Execution Rule

Default approach:

- Preserve strengths
- Improve weak points
- Refactor carefully
- Ship cleaner versions
- Respect existing intent

Never destroy good foundations chasing novelty.

---

# Final Non-Negotiable

A user opening Endeavrly should still feel:

> This feels calm.  
> This feels smart.  
> This helps me move forward.

If a change weakens that feeling, it is the wrong change.
